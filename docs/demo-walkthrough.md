# 전 과정 데모·시뮬레이션 가이드

주문 접수 → 벤더 제작 → 배송 완료까지, 고객·벤더·운영자 세 역할을 오가며
어드민 최소 개입 자동화 플로우를 직접 돌려보는 스크립트.

마지막 검증: 2026-06-13, 브라우저에서 15단계 전 구간 완주 (DM-000004).

---

## 0. 준비

### 앱 띄우기
```bash
npm run dev          # http://localhost:5173
```

### 데모 계정 (비밀번호 모두 `demo1234`)
로그인 페이지(`/login`) 하단의 "둘러보기" 버튼으로 한 번에 전환할 수 있다.

| 역할 | 이메일 | 표시 이름 |
|------|--------|-----------|
| 운영자 | admin@demo.com | Operations |
| 벤더(메인) | supplier@demo.com | SUPPLIER-CN-01 |
| 고객 | customer@demo.com | Jiwon Kim |

> 메인 벤더 1곳뿐이므로 모든 주문이 `SUPPLIER-CN-01`로 자동 매칭된다
> (`settings.defaultSupplierId`). 데모는 이 벤더만 쓰면 된다.

### 초기화 (데모 반복용)
운영자 → **설정** → 빨간 "DB 초기화" 버튼.
시드 상태(주문 DM-000001·DM-000002 + 후보·견적)로 되돌아간다.
완전 초기화가 필요하면 브라우저 콘솔에서 `localStorage.clear()` 후 새로고침.

## 데모의 핵심 메시지

**운영자가 손대는 곳은 단 3번뿐**이고, 나머지는 전부 자동으로 다음 역할에게 넘어간다.

- ① 디파짓 입금 확인
- ② 잔금 입금 확인
- ③ 실물 수령 확인

(결제 자체는 앱 밖 송금이라, 입금 확인만 사람이 누른다.)

---

## 시나리오 A — 새 주문 풀 플로우 (권장 데모)

세 역할을 오가며 15단계를 완주한다. 각 단계 끝의 **자동**이 핵심 포인트.

### 1. 🟢 고객 — 주문 제출
- `/custom/new` 접속 (또는 헤더 "주문제작")
- 스타일: **RING-001 — 솔리테어 링 (6프롱)**, 링 사이즈 `6 US`, 배송국가 `USA`
- 레퍼런스 사진 한 장 선택(예: "밴드"), 약관 동의 → **요청 제출**
- → Order ID(예: `DM-000004`)와 조회 코드 발급. **이 코드를 메모**(고객 포털 재진입용).
- **자동**: 메인 벤더에게 다이아 후보 태스크 발행, 레퍼런스 즉시 전달.

### 2. 🔵 벤더 — 다이아 후보 제출
- `/login` → "서플라이어로 둘러보기"
- 큐 맨 위 **다이아 후보** 태스크 열기 (브리프에 "1.5ct round · E/VS1…"가 자동 생성돼 있음)
- IGI 번호 `LG599000111`, 캐럿 `1.5`, 공급가 `520`, 샘플 이미지 선택 → **제출**
- **자동**: 완결성 통과 → 벤치마크 자동가($1,123)로 고객에게 즉시 공개.
  - 검증: 벤더 화면에 고객 이름·Order ID·판매가가 **안 보임** (PR-xxxxx로만 식별).

### 3. 🟢 고객 — 스톤 선택
- `/track/DM-000004?code=발급코드` 접속
- 체크포인트 ① 후보에서 **이 스톤 선택**
- **자동**: 후보 낸 벤더에게 재고확인 태스크 발행.

### 4. 🔵 벤더 — 재고 확인
- 서플라이어 큐 → **재고 확인** 태스크 → **재고 있음 — 이 주문에 홀드**
- **자동**: 다이아 자동 락 + (스펙 있으면) **견적 자동 생성·발송**.

### 5. 🟢 고객 — 견적 수락
- 포털에서 견적(센터스톤·메탈·세팅·합계·디파짓·잔금) 확인 → **견적·약관 수락**
- **자동**: 운영자 체크리스트에 "디파짓 확인 대기" 표시.

### 6. 🟠 운영자 — 디파짓 확인 ①
- `/login` → "운영자로 둘러보기" → 주문 관제 → 해당 주문
- 견적 빌더의 **디파짓 수령 → CAD** 클릭
- **자동**: CAD 단계 전환 + 벤더에게 CAD 태스크 발행.

### 7. 🔵 벤더 — CAD 제출
- 서플라이어 큐 → **3D CAD** 태스크 → 360°/측면 슬롯에 샘플 선택 → **제출**
- **자동**: 고객 포털 체크포인트 ②(디자인 승인) 활성화.

### 8. 🟢 고객 — 수정 요청 (핀)
- 포털 체크포인트 ② → **수정 요청 (핀 찍기)** → CAD 이미지 클릭해 핀 → 칩 선택(예: "더 얇게") → **수정 요청 보내기**
- **자동**: 벤더에게 핀이 동봉된 새 CAD 태스크 발행. (무료 1회, 이후 디자인비 $15 자동 가산)

### 9. 🔵 벤더 — CAD V2 제출
- 새 CAD 태스크에 "CAD V1 — 마이너 수정 요청됨" + 고객 핀이 표시됨
- 슬롯 채우고 **제출**.

### 10. 🟢 고객 — 디자인 승인
- 포털에서 V2 확인 → **승인 — 제작 시작**
- **자동**: PRODUCTION 전환 + 벤더에게 QC 태스크 발행.

### 11. 🔵 벤더 — QC 제출
- **최종 QC 파일** 태스크 → 영상·인증서 샘플 선택 + 실중량 입력(예: `4.35`) → **제출**
- **자동**: 실중량 자동 정산(잔금 $809→$823) + 고객에게 최종 컨펌 액션 생성.

### 12. 🟢 고객 — 최종 실물 컨펌
- 포털 체크포인트 ③ "이 영상 속 실물이 그대로 배송됩니다" → **컨펌 — 배송 시작**
- **자동**: 잔금 대기(BALANCE) 전환.

### 13. 🟠 운영자 — 잔금 확인 ②
- 주문 관제 → 해당 주문 → **잔금 수령 → 배송 태스크**
- **자동**: 벤더에게 수령지 주소가 담긴 배송 태스크 발행.

### 14. 🔵 벤더 — 운송장 제출
- **운영자 배송** 태스크 → 운송장 번호 입력 → **제출**
- **자동**: SHIPPING 전환 + 고객 포털에 트래킹 번호 표시.

### 15. 🟠 운영자 — 수령 확인 ③
- 주문 관제 → 해당 주문 → **수령 완료 — 배송 처리**
- → 주문 **배송완료(DELIVERED)**.

## 시나리오 B — 빠른 데모 (시드 주문 활용, 5분)

새 주문을 안 만들고 시드에 이미 있는 주문으로 특정 구간만 보여줄 때.

- **DM-000001** (스톤 선택 단계): 후보 2개 공개됨. 고객 포털
  `/track/DM-000001?code=QX7K-M9P2`에서 스톤 선택부터 시작 → 시나리오 A의 4단계로 이어감.
- **DM-000002** (CAD 단계): CAD V1 발행 상태. 포털
  `/track/DM-000002?code=H3WT-8RVK`에서 디자인 승인/수정부터 시작.

> 조회 코드는 운영자 주문 관제의 각 주문 상세 상단, "조회 코드"에 표시된다.

## 데모 중 강조할 검증 포인트

1. **보안 격리** — 벤더 화면엔 고객 신원·Order ID·판매가가 전혀 없다 (PR-xxxxx·IGI 번호로만).
   고객 화면엔 원가·벤더·멀티플라이어가 없다.
2. **자동 가격** — 후보 제출 즉시 벤치마크 자동가로 공개 (운영자 가격 입력 불필요).
3. **수정 루프** — 고객 핀이 글자 없이 그대로 벤더에게 전달, 같은 핀이 벤더 언어로 렌더.
4. **실중량 자동 정산** — QC 제출 시 잔금이 자동 가감, 운영자 계산 없음.
5. **감사 로그** — 주문 상세 맨 아래에 전 단계 actor·시각이 기록 (자동 액션은 `auto`).
6. **모니터링** — 운영자 대시보드 미디어 피드에서 오간 사진·영상을 사후 확인, 문제 시 숨김.

## 자동화/시뮬레이션 (선택)

지금은 사람이 클릭하며 시뮬레이션한다. 더 자동화하고 싶으면 두 가지 선택지:

- **스토어 레벨 시나리오 테스트** — `src/lib/__tests__/autoFlow.test.js`가 이미
  인테이크→…→DELIVERED 전 구간을 코드로 한 번에 돌린다. `npm test`로 실행.
  새 시나리오를 추가하면 클릭 없이 검증 가능.
- **시드에 "데모 진행 단계" 주문 추가** — 특정 단계에서 시작하는 주문을
  `seed.js`에 더 넣으면, 초기화 후 바로 그 지점부터 데모할 수 있다 (요청 시 추가).

> UI를 자동으로 클릭하는 E2E 스크립트(Playwright)로 만들 수도 있다. 필요하면 말해주세요.
