# 어드민 최소 개입 자동 운영 플로우 (브레인스토밍 확정)

날짜: 2026-06-12 · 상태: 승인됨

## 목표

어드민 루틴 개입을 3가지로 축소: **① 디파짓 입금 확인 ② 잔금 입금 확인 ③ 실물 수령 처리**.
고객 주문 → 벤더 태스크 → 벤더↔고객 시각 자료 커뮤니케이션 → 최종 컨펌 → 벤더 제작·운영자 배송까지
전 구간을 자동 연결한다. 어드민 화면은 "승인하는 곳"에서 "감시하는 곳"으로 전환.

## 확정된 결정

| 결정 | 선택 |
|---|---|
| 다이아 소비자가 | 벤치마크 자동가 (9×7 매트릭스 단가 × 캐럿 × 멀티플라이어) |
| 시각자료 검수 | 즉시 전달 + 사후 모니터링 (미디어 피드 + 숨김 버튼) |
| 벤더 매칭 | `style.supplierId ?? settings.defaultSupplierId` — 현재는 전 주문이 기본 벤더 1곳으로 |
| 구현 구조 | 기존 PR(벤더 태스크)·체크포인트·핀 구조 유지, 단계 사이를 자동 발행으로 연결 |

## 자동 연결 체인

1. **인테이크 제출** (`createIntake`)
   - 솔리테어 → `diamondCandidates` PR 자동 발행 (브리프는 스톤 선호에서 생성, 마감 +`autoDueDays`, 배치 유효 +`batchValidDays`)
   - 멀티/스타일 주문 → 승인 스펙(스타일+메탈) 있으면 즉시 자동 견적 생성·발송, 없으면 `weightLabor` PR 자동 발행
   - 레퍼런스 미디어는 `approved`로 즉시 벤더 전달 (사전 승인 게이트 제거)
2. **벤더 후보 제출** (`submitCandidates`) → 완결성(IGI·캐럿·이미지) + 벤치마크 존재 시 자동가로 즉시 공개, 아니면 보류(체크리스트 표시)
3. **고객 선택** → 재고확인 PR (기존) → 벤더 확인 → 자동 락 (기존) → **자동 견적 생성·발송** (`tryAutoQuote`: 다이아 원가는 후보, 공임·중량은 스펙)
4. **견적 수락** → 어드민 입금 확인(`markDepositReceived`) → **CAD PR 자동 발행**
5. **고객 수정 요청** (`decideCad` minorRevision) → **새 CAD PR 자동 발행** (핀 동봉, 무료 한도·디자인비 기존 로직 유지)
6. **고객 승인** (`decideCad` approved) → PRODUCTION (기존) + **QC PR 자동 발행**
7. **벤더 QC 제출** (`submitQcForPr`) → 실중량 자동 정산(`recordActualWeight` 자동 호출) + 최종 컨펌 액션 (기존)
8. **고객 최종 컨펌** → BALANCE (기존) → 어드민 잔금 확인(`markBalanceReceived` 신설) → **ship PR 자동 발행** (신규 태스크 타입)
9. **벤더 운송장 제출** (`submitShipment` 신설) → 배송 마일스톤 자동 갱신 + SHIPPING
10. **어드민 수령 확인** (`markOrderDelivered` 신설) → DELIVERED + 아카이브 마일스톤

## 가드·예외 처리

- `selectCandidate` 스토어 레벨 가드: 비공개·품절·배치 만료 후보 선택 거부
- 배치 만료 스윕(`sweepExpiredBatches`): 만료 PR 종료 + 미판매 후보 자동 비공개 — 포털/벤더 큐/체크리스트 진입 시 lazy 실행
- 자동화 실패(기본 벤더 미설정, 벤치마크 누락, 스펙 없음)는 주문을 멈추지 않고 보류 + 체크리스트 표시
- 모든 자동 액션은 감사 로그에 actor `auto`로 기록

## 어드민 = 모니터링

- **미디어 피드** (`mediaFeed`): 레퍼런스/CAD/QC 시각자료 시간순, `hideMedia`로 숨김
  - 레퍼런스 숨김 → 벤더 브리프에서 제외
  - CAD 숨김 → 포털에서 제외 + 해당 CAD PR 재오픈(벤더 재제출)
  - QC 숨김 → 제출 철회 + QC PR 재오픈 + 최종 컨펌 액션 취소
- 데일리 체크리스트 추가 항목: 디파짓 확인 대기, 잔금 확인 대기, 보류 후보
- 역할별 헤더 배지: 벤더(열린 태스크), 고객(대기 액션), 어드민(결제 확인 대기)
- 수동 개입 능력은 전부 유지 (PR 수동 발행, 수동 가격, 수동 마일스톤)

## 바뀌지 않는 것

보안 프로젝션(벤더에 고객 신원·판매가·Order ID 미노출, 고객에 원가·벤더·멀티플라이어 미노출),
익명 핀 주석, 감사 로그, 무료 수정 한도·디자인비, 견적 스냅샷.

## 테스트

자동 연결 체인 전체(인테이크→PR, 락→자동견적, 수정→재발행, 입금→CAD, 잔금→ship, 운송장→SHIPPING),
자동가 책정, 보류 후보, 선택 가드, 배치 만료, 미디어 피드/숨김. 기존 57개 테스트 중
의도적 동작 변경(자동 견적 버전, 레퍼런스 즉시 승인)에 해당하는 단언만 갱신.
