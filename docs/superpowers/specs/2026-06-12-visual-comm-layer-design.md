# 비주얼 커뮤니케이션 레이어 — 사진/영상 중심 벤더↔구매자 플로우

날짜: 2026-06-12 / 상태: **확정** (브레인스토밍 세션에서 사용자 승인)
배경: 벤더(중국어)와 구매자(EN/中文/KO/ES)가 언어가 달라 텍스트 대화가 불가능. 플랫폼이 중간에서 구매자의 스타일 의도를 시각 자료로 벤더에게 전달하고, 벤더는 사진/영상으로 응답, 구매자가 컨펌하는 **무언어(zero-language) 커뮤니케이션 레이어**를 기존 ops 도메인 위에 얹는다.

## 0. 확정된 결정 (브레인스토밍)

| 결정 | 선택 | 비고 |
|---|---|---|
| 벤더 참여 모델 | 운영팀이 주문당 1곳 지정 | 기존 ops 스펙(PR 배정)과 일치. 입찰/마켓플레이스 아님 |
| 컨펌 체크포인트 | 3단계: 스톤 → 디자인 → 최종 | 맞춤 제작 = 환불 불가이므로 비싼 실수를 싼 단계에서 차단 |
| 피드백 방식 | 사전 번역 칩 + 이미지 핀 주석 | 자유 텍스트 입력 없음 — 오역·연락처 교환 구조적 차단 |
| 구매자 화면 구조 | 타임라인 포털형 | "지금 할 일" 액션 카드가 펼쳐지는 세로 타임라인. ClientPortal 확장 |
| 의도 입력 | 카탈로그 베이스 + 레퍼런스 사진 + 핀 주석 | 핀+칩 문법을 입력~컨펌 전 구간에서 통일 |

## 1. 전체 플로우

```
구매자: 베이스 스타일 선택 + 레퍼런스 사진 업로드 + 핀 주석
  → 운영팀 검수 (레퍼런스 적합성: 도용/연락처 포함 이미지 차단)
  → 벤더 브리프 = PR (베이스 스타일 + 검수 통과 레퍼런스 + 핀 주석 中文 렌더링)

체크포인트 ① 스톤  : 벤더가 실물 다이아 사진/영상 후보 제출 → 운영 검수 → 구매자 선택 (기존 diamondCands 플로우)
체크포인트 ② 디자인: 벤더가 CAD 렌더링 제출 → 구매자 승인 or 핀+칩 수정 요청 → 버전 V1→V2→… (기존 cadReviews)
체크포인트 ③ 최종  : 벤더가 완성품 QC 사진/영상 제출 → 구매자 최종 컨펌 → 잔금/배송 (기존 milestones + customerActions)
```

새 도메인이 아니라 **기존 ops 도메인(13 마일스톤·PR·cadReviews·customerActions)의 UI 강화 레이어**다.

## 2. 핵심 컴포넌트

- **`<PinAnnotator>`** — 이미지 위 핀 + 칩 선택 + 수치 스테퍼.
  - 데이터: `{pinId, x, y, part, chipKey, value?}` — 좌표는 % 기반 (반응형).
  - 의도 입력 단계와 디자인 수정 요청 단계에서 동일 컴포넌트 재사용.
  - 읽기 전용 모드: 벤더/운영 화면에서 같은 핀을 해당 언어로 렌더링.
- **칩 어휘 사전 (`chipCatalog`)** — 부위(밴드/프롱/스톤/체인/클래스프/갤러리/마감) × 수정 유형(두께/높이/개수/마감/색조/형태) 매트릭스.
  - 전 항목 key 기반, 4개 언어(EN/中文/KO/ES) 렌더링. 구매자 "더 얇게" = 벤더 "更细" = 같은 `band.thinner` key.
  - 자유 텍스트 입력은 어디에도 없음.
- **`<MediaCheckpoint>`** — 타임라인 안에서 펼쳐지는 액션 카드. 미디어 갤러리(사진/영상 슬롯) + [승인] / [수정 요청] 버튼 쌍. 세 체크포인트가 같은 골격.
- **`<CompareView>`** — 체크포인트 카드 내부의 "내 레퍼런스 vs 벤더 제안" 좌우 대조 뷰.

## 3. 데이터 모델 (mock store 확장)

```
intakes        += referenceMedia[{id, url, status[pending|approved|rejected], annotations[{pinId,x,y,part,chipKey,value?}]}]
cadReviews     .feedback → 구조화 주석 배열 [{pinId,x,y,part,chipKey,value?}] (버전당 1레코드, 덮어쓰기 금지 유지)
chipCatalog    (key, part, kind, labels{en,zh,ko,es}, valueType[none|mm|count|enum], 신설 — 시드 + 어드민 편집)
customerActions type 추가: finalConfirmation (최종 실물 컨펌, 증거 보존)
```

## 4. 화면

### 구매자
- **의도 입력** (`/custom/new` 확장): 카탈로그 베이스 선택 → 레퍼런스 사진 업로드(선택) → `<PinAnnotator>`로 "이 부분은 이 사진처럼" 표시 → 제출.
- **주문 타임라인** (`/portal/:orderId`): 세로 타임라인, 완료 단계 접힘, 내 차례 단계만 `<MediaCheckpoint>` 카드로 펼쳐짐. 다음 단계 예상일 표시.
- **수정 요청 화면**: 렌더링 이미지 위 `<PinAnnotator>` + 칩 팔레트 + 수치 스테퍼 + "무료 수정 N회 남음" 카운터.
- **최종 컨펌**: QC 영상 + "이 영상의 실물이 배송됩니다" 고지 + 컨펌 버튼 (증거 보존).

### 벤더 (`/supplier` 확장)
- 태스크 카드 = 브리프: 베이스 스타일 + 검수 통과 레퍼런스 + 핀 주석(中文 렌더링) + 마감일.
- **슬롯형 제출**: 렌더링 360°/측면/착용 등 정해진 슬롯에 업로드 — 필요한 각도를 언어 설명 없이 슬롯 구조가 강제.
- 고객 신원·판매가·마진 미노출 (기존 보안 경계 유지).

### 운영
- 레퍼런스 검수 큐 (승인/반려), 칩 카탈로그 관리, 수정 요청 중계 현황. 기존 AdminOpsOrder에 통합.

## 5. 가드레일

- 마이너 수정 1회 무료 → 이후 `designChangeFeeUsd` 부과 (기존 설정 재사용). 무한 수정 루프 방지.
- 핀 주석은 버전 귀속 (V2의 핀은 V2에 고정, 덮어쓰기 금지 — cadReviews 원칙 동일).
- 레퍼런스는 운영 검수 통과분만 벤더 전달.
- 최종 컨펌 증거(타임스탬프·영상 링크) 보존 — 분쟁 방어.
- 모든 시각 데이터에 자유 텍스트 필드 없음 — `maskContacts` 우회 경로 차단.

## 6. Non-goals

실시간 영상통화, 자유 텍스트 자동번역, 벤더 다중 입찰, 3D 뷰어, AI 스타일 추천, 실제 파일 업로드 백엔드(MVP는 mock store URL).
