# BeloveD 어드민 운영 가이드

커스텀 주문은 고객에게 복잡한 제작 단계를 모두 보여주기보다, 운영자가 확정 제안 1건과 몇 번의 핵심 컨펌만 관리하는 방식으로 운영합니다.

## 핵심 원칙

모든 주문은 아래 고객 확인 흐름으로 관리합니다.

1. **확정 제안**: 운영자가 벤더 상담 후 제안 스톤을 지정하고(후보 테이블의 "제안 스톤으로 지정"), 확정 스펙 + 총액 견적 1건을 보냅니다. 고객에게는 미디어·스펙·총액만 보이고 breakdown(다이아/메탈/패키지)은 어드민 전용입니다. 동급 대체 정책이 제안 카드에 함께 표시됩니다.
2. **디파짓**: 고객이 제안을 컨펌하면 Zelle/Venmo 송금 안내가 열립니다. 고객이 "보냈어요"를 누르면 어드민 워크벤치에 송금 보고 뱃지가 뜨고, 운영자가 입금을 확인하면 스톤 락 + CAD 단계가 자동 시작됩니다.
3. **디자인 승인**: 운영자가 CAD, 렌더, 디자인 시안을 올리고, 고객이 승인하거나 수정 요청을 남깁니다.
4. **완성품 컨펌**: 운영자가 최종 QC 사진·영상을 올리고, 고객 확인 후 잔금(같은 Zelle/Venmo 카드)·배송으로 넘어갑니다.

고객 포털의 다이아 후보 비교·선택 화면은 제거되었습니다. 후보 검수·공개·가격은 어드민 내부 도구로만 사용합니다. Zelle/Venmo 계정은 **Admin → Settings**에서 편집합니다.

## Order Control 사용법

**Admin → Order Control**에서 주문 로우 아무 곳이나 클릭하면 주문 상세 워크벤치로 이동합니다.

상세 화면은 아래 순서로 사용합니다.

- **고객 확인 대기**: 지금 고객이 해야 할 일이 있는지 가장 먼저 확인합니다.
- **고객 진행 흐름**: 고객에게 보이는 3단계 상태를 간단히 업데이트합니다.
- **다음 고객 컨펌 보내기**: 다이아 후보, 디자인 시안, 완성품 QC 미디어를 업로드합니다.
- **상담 채팅**: 웹 상담과 인스타·Threads·커뮤니티 DM 내용을 주문별로 기록합니다.
- **주문 요약**: 고객이 제출한 요청 정보를 확인합니다.
- **내부 관리**: 운영자만 보는 상태와 내부 메모를 관리합니다.

## 고객 공개 업데이트

**고객 진행 흐름** 카드마다 아래 항목을 조정할 수 있습니다.

- **상태**: 내부 진행 상태입니다.
- **고객 업데이트**: 고객 포털에 짧게 보여줄 문장입니다.
- **고객에게 공개**: 켜면 고객 포털의 3단계 상태 카드에 반영됩니다.

긴 설명이나 실제 선택 자료는 여기에 쓰지 말고, **다음 고객 컨펌 보내기**에 미디어로 업로드합니다.

## 업로드 규칙

- 다이아 후보: 후보 1개당 사진·영상 최대 5개
- 디자인 승인: CAD, 렌더, 사진, 영상 최대 5개
- 완성품 컨펌: 최종 QC 사진·영상 최대 5개
- 큰 이미지는 데모 저장 전에 클라이언트에서 압축됩니다.

## 고객 포털

고객은 기술적인 전체 마일스톤 대신 아래 3단계만 봅니다.

- 스톤
- 디자인
- 완성품

고객은 현재 필요한 컨펌, 업로드된 미디어, 상담 채팅, 운영자가 공개한 짧은 업데이트만 확인합니다.

## 운영 메모

- 가격 범위와 벤치마크는 **Diamond Pricing**에서 관리합니다.
- 샘플 디자인, 서브카테고리, 사진·영상은 **Style Library**에서 관리합니다.
- 인스타그램, Threads, 미국·중국 커뮤니티 문의는 API 연동 전까지 **상담 채팅**에 주문별로 기록합니다.
- 기술 마일스톤 요약은 감사·디버깅용입니다. 일상 운영은 3단계 고객 컨펌 흐름으로 처리합니다.
